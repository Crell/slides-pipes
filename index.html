Closures


         $c = 5;

         $fn = function ($a, $b) use ($c) {
           return ($a * $b) * $c;
         };

         print $fn(2, 3); // 25
  • Didn't have primitive types or return types yet

Closures with references


         $c = 5;

         $fn = function ($a, $b) use (&$c) {
           return ($a * $b) * $c;
         };

         $c = 2;

         print $fn(2, 3); // 10

How closures really work


         $c = 5;

         class Closure_abc123 extends Closure {
           public function __construct(private $c) {}

           public function __invoke(int $a, int $b) {
             return ($a * $b) * $this->c;
           }
         }
         $fn = new Closure_abc123($c);

         print $fn(2, 3); // 25
  • This is what the compiler actually does internally.
  • __invoke lets any object get called directly.

DIY


         $c = 5;

         class Computation {
           public function __construct(private $c) {}

           public function __invoke(int $a, int $b) {
             return ($a * $b) * $this->c;
           }
         }

2025

PHP 8.5: Now that we're all caught up...

Pipes

This is fugly, but common


        $arr = ['A', 'B', 'C', 'D', 'E'];

        $result = array_values(array_filter(is_odd(...), array_map(fn(string $x): bool => ord($x), $arr)));
  • This is gross. Raise your hand if you've done it.

This is better, but still clumsy


          $arr = ['A', 'B', 'C', 'D', 'E'];

          $temp = array_map(fn(string $x): bool => ord($x), $arr));
          $temp = array_filter(is_odd(...), $temp);
          $temp = array_values($temp);
          $result = $temp;
  • Very subtley different from previous, if any provided args are inlined function calls
  • Isn't there an easier way to do this?
  • Let's automate away $temp

Much nicer


          $arr = ['A', 'B', 'C', 'D', 'E'];

          $result = $arr
            |> (fn($x) => array_map(ord(...), $x))
            |> (fn($x) => array_filter(is_odd(...), $x)
            |> array_values(...)
          ;
  • Compiles to the same as previous slide
  • Optimizes away extra closures where it can
  • Much easier to see what's going on
  • Need parens around inline closures to not confuse the parser

|>

  • Read as "pipe to" or "pass result to"
  • Takes only single-param callables
  • The more steps there are, the more it helps

Partial application with higher order functions


        // ...

        function sorted(\Closure $sorter): \Closure {
          return fn(array $a) => usort($a, $sorter);
        }
        

          $result = scandir('.', SCANDIR_SORT_NONE)
            |> (fn($v) => array_filter($v, fn($x) => str_contains($x, 'page')))
            |> (fn($v) => array_map(fn($x) => new SplFileInfo($x))
            |> sorted(size_sort(...))
            |> array_reverse(...)
            |> (fn($v) => array_chunk($v, 5)[0])
            |> (fn($v) => array_map(fn(SplFileInfo $x) => sprintf('%d %s %s, $x->getSize(), $x->getOwner(), $x->getFilename()))
          ;
  • This is looking nicer
  • sorted() now returns a single-param closure, which pipes can use directly
  • We are "partially" "applying" arguments to usort()
  • sorted() is "higher order", because it takes/returns other functions

Partial application is easy


        // ...

        function take(int $count): \Closure {
          return fn(array $v) => array_chunk($v, $count)[0];
        }
        

          $result = scandir('.', SCANDIR_SORT_NONE)
            |> (fn($v) => array_filter($v, fn($x) => str_contains($x, 'page')))
            |> (fn($v) => array_map(fn($x) => new SplFileInfo($x))
            |> sorted(size_sort(...))
            |> array_reverse(...)
            |> take(5)
            |> (fn($v) => array_map(fn(SplFileInfo $x) => sprintf('%d %s %s, $x->getSize(), $x->getOwner(), $x->getFilename()))
          ;
  • Do the same for grabbing just a certain number of values from the front.
  • take() is a common standard name for this

Higher order functions are so simple


        // ...

        function afilter(\Closure $filter): \Closure {
          fn($v) => array_filter($v, $filter);
        }
        

          $result = scandir('.', SCANDIR_SORT_NONE)
            |> afilter(fn($x) => str_contains($x, 'page'))
            |> (fn($v) => array_map(fn($x) => new SplFileInfo($x))
            |> sorted(size_sort(...))
            |> array_reverse(...)
            |> take(5)
            |> (fn($v) => array_map(fn(SplFileInfo $x) => sprintf('%d %s %s, $x->getSize(), $x->getOwner(), $x->getFilename()))
          ;
  • Another simple one, though doesn't fix everything

You can quickly make a lot of them


        // ...

        function amap(\Closure $mapper): \Closure {
          return fn(array $v) => array_map($mapper, $v);

Self-describing code with pipes


        // General purpsoe
        function sorted(\Closure $sorter): \Closure {}
        function take(int $count): \Closure {}
        function afilter(\Closure $filter): \Closure {}
        function amap(\Closure $mapper): \Closure {}

        // Task specific
        function size_sort(SplFileInfo $a, SplFileInfo $b): int { }
        function format_file(\SplFileInfo $f): string {}
